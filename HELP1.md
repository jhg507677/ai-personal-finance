# Oauth란
제 3의 서비스에 계정 관리를 맡기는 방식
리소스 오너 : 인증 서버에 자신의 정보를 사용하도록 허가하는 주체, 서비스를 이용하는 사용자
리소스 서버 : 리소소 오너의 정보를 가지며 리소스 오너의 정보를 보호하는 주체, 네이버, 구글 등
인증 서버 : 클라이언트에게 리소스 오너의 정보에 접근할 수 있는 토큰을 발급하는 역할을 하는 애플리케이션
클라이언트 애플리케이션 : 인증 서버에게 인증을 받고 리소스 오너의 리소스를 사용하는 주체, 현재 가계부 서버

권한 부여 승인 타입: 권한에 접근할 수 있는 코드와 리소스 오너에 대한 액세스 토큰을 발급 -> 사용자 데이터가 외부로 전송되지 않아 안전
인증서버에 요청을 처음 보내는 경우 사용자에게 보이는 페이지를 로그인 페이지로 변경하고 사용자의 데이터에 접근 동의를 받음, 최초 1회만 진행
이후에는 인증 서버에서 동의 내용을 저장하고 있기때문에 로그인만 진행

액세스 토큰 응답이란?
인증 코드를 받으면 액세스 토큰으로 교환
